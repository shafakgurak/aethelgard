Architecture Deep Dive

Aethegard's Air-Gap Is Your Security Moat

In regulated sectors, the training platform itself must not become a liability. Æthelgard eliminates the risk entirely.

🛡 Air-Gap Security Architecture

Every major cloud-based training platform transmits learner names, scores and session data to external servers. This creates GDPR obligations, data processor agreements, and a permanent external dependency. If their servers go down, your training stops.

Æthelgard transmits nothing. All logic — questions, scoring, certificate generation and cryptographic token validation — runs entirely inside the browser. There is no server. There is no account. There is no cloud bill.

This is especially valuable for schools with inconsistent internet access, healthcare settings with strict data governance, defence contractors, legal firms, and any regulated environment where a Data Protection Officer would otherwise block a SaaS deployment.

🔐

Token-Based Auth

AETH- keys validated client-side using expiry timestamps. No server round-trip. Tokens expire automatically.

📜

GDPR Scroll-Gate

Built-in EULA must be fully scrolled and accepted before access is granted. Consent is recorded in session.

🛡

Anti-Tamper Layer

Right-click disabled. DevTools blocked. Print-screen watermarked with institution name. Source obscured.

🎓

Shard Certification

Binary shard IDs are encoded with date and token prefix. Certificates are verifiable and cannot be forged without the deployment key.
